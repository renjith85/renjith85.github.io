Secure copy protocol(SCP) allows files to be copied between machines/servers in a secure way.

Usage

scp <source-specification> <destination-specification>

Tip:
  • Format for a resource in remote(source or destination) is: user@machine:filename
  • Format for file specification on local system is : filename
  • While accessing a remote server
    • make sure user account exists in remote destination
    • make sure filesystem has sufficient permission to be accessed by user account

Examples:

  • Copy a file(say "sample.adoc") from local to remote system(User is in local system shell)

scp sample.adoc remote-user@<machine-ip>:file-location

  • Copy a file(say "sample.adoc") from remote system onto local system current folder(User is in local system shell)

scp remote-user@<machine-ip>:<absolute-location-of-sample.adoc> .

  • Copy a remote system file onto local system tmp folder(User is in local system shell)

scp remote-user@<machine-ip>:<location-of-remote-file> /tmp

SCP using ssh key

In most of the scenarios, you will be required to use a certificate(.pub or .pem or ssh) key to access the underlying servers.

scp -i <identity-file> <source-specification> <destination-specification>

Important:

Make sure your identity file has only limited permission(chmod 400 <identity-file>, if not some servers will complain the access key is too open
