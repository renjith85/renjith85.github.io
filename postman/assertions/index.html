Postman Response Assertions

Within postman, test assertions can be included to check the validity of the API request and response objects. Postman test assertions follows a BDD style and are based on the capabilities of the Chai Assertion Library. Read more about Chai at https://www.chaijs.com/api/bdd/

Assert status
pm.test("Must fail with response status", function () {
    pm.response.to.have.status(400);
});
Assert json response body element
pm.test("Verify jwt issuer", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql("CODE-001");
});
Assert absence of a json element
pm.test("Verify if users has an entitlement", function(){
    var data = pm.response.json();
    pm.expect(data).to.not.have.ownPropertyDescriptor('entitlements');
})
Check if an integer data value is above a specific value
pm.test("Verify if users exists", function(){
    var data = pm.response.json();
    pm.expect(data.paging.total).to.be.above(1);
})
Check integer equality
pm.test("Verify if users exists", function(){
    var data = pm.response.json();
    pm.expect(data.paging.total).to.be.equal(0);
})
Decode a JWT

There will be scenarios in which you want to check the validity of a Bearer token represented as a JWT token

As you would be familiar JWT tokens, it is a concatenated string of Base64 encoded representation of JWT header, payload(aka claims) and signature.We define a simple function to do a split based on JWT delimiter(.) and then pick the payload(claims) value and decode the Base64 value using atob function in javascript. Then the structure can be parsed using a JSON object parser function.

function jwt_decode(jwt) {
  var parts = jwt.split('.'); // header, payload, signature
  return JSON.parse(atob(parts[1]));
}
